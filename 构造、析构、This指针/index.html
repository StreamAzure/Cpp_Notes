    类的基本定义

    设计原则：尽可能将公有接口和实现细节分开，即封装。数据隐藏，即将数据放在类的private部分，是一种封装。把类函数定义和类声明放在不同的文件中也是一种封装。

    class Student{
public:
    string name;
    int age;
    void show(){ // 直接在类声明中定义了 show 函数，它其实是内联函数
        cout << name << " " << age << endl;
    }
};

    类成员与访问控制

    访问控制关键字

    • public：使用类对象的程序可以直接访问
    • private：只能通过类的成员函数或友元函数访问
    • protected：只能通过类的成员函数和子类访问，其他不行。

    当一个成员函数/数据成员没有声明在某个访问控制关键字下时，默认为private。因此通常来说不必显式使用private。

    类的大小

    • char为1字节，short为2字节，int为4字节，double为8字节。

    • 当类的属性中存在多种类型的变量时，由于存在内存对齐，变量声明顺序会影响类的大小。

    • 如果一个属性都没有，只有成员函数，由于成员函数不占类的大小，类的大小为1字节作为占位。

    • 但如果方法是虚函数，由于虚函数本身是个指针，占4字节。（详见“多态”章）

    class base1{
private:
    char a;
    int b;
    double c;
};
class base2{
private:
    char a;
    double b;
    int c;
};

    构造函数与析构函数

    构造函数

    基本写法

    class Student {
public:
    int id;
}
    • 默认构造函数：
    // 写法1：
Student(){
  id = 0;
}
// 写法2：
Student() : id(0) {}
    • 含参构造函数
    // 写法1：
Student(int _id){
    id = _id;
}
// 写法2：
Student(int _id) : id(_id) {}
    • 拷贝构造函数
    Student(const Student& s) {
  //const表示s的值不能被修改
  //即保证被拷贝对象的值不会被修改
  id = s.id;
}

    其他细节

    • 构造函数是可重载的，即可以有若干个含参构造函数。
    class Student {
public:
    int id;
    Student(int a) {
        id = a;
    }
    Student(int a, int b) {
        id = a + b;
    }
};
    • 构造函数的手动定义与自动定义
    • 如果没有定义任意一个构造函数，编译器会自动定义三个函数：默认构造函数、拷贝构造函数、析构函数
    • 但如果已经定义含参构造函数或拷贝构造函数的任意一个，其他构造函数不会被自动定义，即此时不能直接调用默认构造函数。
    • 解决方法：无论什么时候都要记得自己定义一个默认构造函数。

    浅拷贝及可能造成的问题

    如果在默认构造函数中，以new为变量buff申请堆上的空间，而后又进行了类对象的拷贝（即调用拷贝构造函数），新的对象会自动申请新的内存空间吗？

    答案是：新对象不会申请内存空间，而是和原对象共用一个buff，即使用的空间是相同的。

    class Teacher {
public:
    int* age;
    Teacher() {
        age = new int(18);
    }
};
int main() {
    Teacher t1; // 原对象 t1，在默认构造函数中 new 了一个 int 大小的空间（即4字节）
    Teacher t2(t1);// 调用自动定义的拷贝构造函数，构建新对象 t2
    cout << *t1.age << endl << *t2.age << endl;
    // 均为18
    cout << t1.age << endl << t2.age << endl;
    // 为同一个地址
    return 0;
}

    以上这种不连带空间申请一起“拷贝”的拷贝方式为浅拷贝。

    浅拷贝得到的对象会在以下情况析构时崩溃！

    class Teacher {
public:
    int *age;
    Teacher() {
        age = new int(18);
    }
    ~Teacher(){
        delete age;
    }
};
void test(){
    Teacher t1;
    Teacher t2(t1);
}
int main(){
    test();
    return 0;
}

    崩溃原因：

    test函数调用结束后，依次释放对象t1和t2，但t1和t2的age共用同一块内存，释放行为会导致这块内存被删除（delete）了两次，从而崩溃。

    深拷贝

    如果希望t2也能有独立的空间存放age，避免上述重复delete的情况，应自行定义拷贝构造函数：

    class Teacher {
public:
    int* age;
    Teacher() {
        age = new int(18);
    }
    Teacher(const Teacher& t){
        age = new int(*t.age);
        //将原对象的age值作为参数，new了新空间，再存储这个参数
    }
};
int main() {
    Teacher t1;
    Teacher t2(t1);
    cout << t1.age << endl << t2.age << endl;
    //输出为两个不同的地址
    return 0;
}

    类对象为另一个类的成员属性

    class Student {
public:
    Student() {};
    Student(const Student& tmp) {}
    ~Student() {};
};

class Teacher {
public:
    Student s; // Teacher的属性中含一个Student类对象
    Teacher() {};
    Teacher(Student& _s) :s(_s) {
        //Teacher的含参构造函数，参数为一个Student对象_s
        //s(_s)即Stucent的拷贝构造函数，将_s拷贝给了s
    }
};

int main() {
    Student stu;
    Teacher t(stu);
    return 0;
}

    当执行main函数中Teacher t(stu);这行代码时，其具体执行流程如下：

    1. 调用Student类的拷贝构造函数，将Student对象stu拷贝给Teacher对象t中的属性s；
    2. 调用Teacher类的含参构造函数，创建一个Teacher对象t。

    析构函数

    用于删除对象。

    特别地，在构造函数中由new关键字申请的内存空间，都需要在析构函数中通过delete关键字对应释放。

    class Student {
public:
    int id;
    ~Student(){}
};

    void fun(Student s) {
    ...
}
Student s1;
fun(Student(s1));
//调用函数时调用了一次拷贝构造函数
//在结束函数调用后自动调用了一次析构函数

    注意这样的析构函数不会释放new出来的（存放在堆中的）内存：

    class Teacher {
public:
    int* age;
    Teacher() {};
    Teacher(int _age) {
        age = new int(_age);
    }
};
int* fun() {
    Teacher t = Teacher(18);
    return t.age;
}
int main() {
    cout << *fun() << endl;
    return 0;
}
//虽然在结束fun的调用时调用了一次析构函数将Teacher对象t释放了，但*t.age没有释放，依然可以正常输出为18

    当Teacher对象t释放后，遗留下来的未能释放的t.age就是所谓的内存泄漏，它所占用的内存要直到程序结束才能被释放。

    要解决这个问题，只需将析构函数改写一下：

    ~Teacher(){
    delete age;
}

    delete能释放new出来的内存。

    This指针与链式调用

    This指针指向调用当前函数的对象。

    使用场景1： 当对象内函数需要返回这个对象时

    class Student {
public:
    int id;
    Student& id_plus1(Student& s) {
        //注意返回类型的区别，这里返回了一个Student对象的引用
        id += s.id;
        return *this;
    }
    Student* id_plus2(Student& s) {
        //注意返回类型的区别，这里返回了一个指向Student对象的指针
        id += s.id;
        return this;
    }
    Student id_plus3(Student& s) {
        //注意返回类型的区别，这里返回了一个Student对象
        id += s.id;
        return *this;
    }
};

    这样的函数可以执行链式调用：

    Student s1(18);
Student s2(18);
s2.id_plus1(s1).id_plus1(s1);
//最后s2.age为54
//此处s2.id_plus1(s1)是个Student对象

    Student s1(18);
Student s2(18);
s2.id_plus2(s1)->id_plus2(s1);
//最后s2.age为54
//注意与上一个代码块的区别，此处s2.id_plus2(s1)是个指针

    Student s1(18);
Student s2(18);
s2.id_plus3(s1).id_plus3(s1);
//最后s2.age为36

    解释一下第三个代码块的结果：

    s2.id_plus3(s1).id_plus3(s1);

    1. 在执行第一个id_plus3时，执行它的对象是s2，s2.id由18变为36；s2.id_plus3(s1)返回的是一个由s2拷贝构造出来的新对象。
    2. 在执行第二个id_plus3时，执行它的是上文所述的新对象，这个对象的id属性由36变为54；
    3. 最后s2.id_plus3(s1).id_plus3(s1);这一整行代码是有返回值的，返回的是一个由上述新对象再次拷贝构造出的第二个新对象，其id属性也是54。

    由上述过程可以看出，s2.id只执行了一次加法，故其值为36.
