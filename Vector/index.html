    Vector

    迭代器

    template <class T, class Alloc = alloc>
class vector{
public:
    tpyedef T           value_type;
    typedef value_type* iterator;
    ...
}

    Vector的迭代器是随机访问迭代器

    Vector的迭代器本质上就是普通指针。例如：

    • vector<int>::iterator ivite;中，ivite的类型是int*；
    • vector<Shape>::iterator svite;中，svite的类型是Shape*

    重要迭代器

    vector的成员迭代器：

    • start：当前使用空间的首个元素，即vec.begin()的返回值
    • finish：当前使用空间的末尾元素，即vec.end()的返回值
    • end_of_storage：当前可用空间的最末位置

    容量

    • vector 实际配置的大小比使用者需求量更大一些，即一个 vector 的容量永远大于或等于其大小。
    • 初始化时，大小=容量=初始化的元素个数
    • 当容量=大小，且有新增元素时，容量翻倍；若翻倍后还不够，就扩张到足够大
    • 容量的扩张过程：实际上就是动态数组扩张的过程，即重新申请内存、移动元素、释放原内存
      • 一旦对 vector 的操作引起容量扩张，则原有的 vector 迭代器全部失效
    • 若使用pop_back()、clear()等删除元素，容量保持不变

    操作

    pop_back()

    • 删除最末尾的元素

    earse(iterator position)

    • 删除由迭代器指定的某个位置的元素
    • 执行删除操作后，迭代器自动自增指向下一个元素。

    clear()

    • 实际调用的是erase()

    insert()

    • 情况一：插入元素不会超出 vector 容量
      1. 如果要移动的元素个数＞插入的元素个数：设插入点迭代器为pos，则从 pos 开始的后面元素都右移 x 个位置，并修改finish迭代器；再在pos位置填充插入的元素。
      2. 如果要移动的元素个数 ≤ 插入的元素个数：先用多出来那些插入元素填充“应该在且可以在”的位置，再将 pos 开始的后面元素“跳跃式”地拷贝到插入元素后的空间；再在pos位置后腾出来的空间填充插入的元素。

    • 情况二：插入元素会超出 vector 容量

      直接申请新的内存空间，pos 开始的后面元素直接拷贝到对应位置，再直接在pos位置填充插入的元素

    面试问题

    vector 中插入一万个数据如何提高效率？再把这一万个数据清掉后内存如何变化？

    • 提前申请好内存：vec.reserve，避免重新分配和数据复制
    • 使用 C++11 特性：emplace_back()，减少对象拷贝和构造次数

    为什么 vector 容量扩张时是 1.5 倍或 2 倍？

    • 参考资料
    • 其实 2 倍不如 1.5 倍好，但至少能够保证 push_back 操作的均摊时间复杂度为 O(1)
    • 1.5 倍是最好的，因为能够增大“重新使用之前释放掉的内存空间”的可能性。如果之间释放掉的空间通过新的分配又被vector再次使用了，且这部分空间对应的 Cache（如TLB等）还未被替换掉，就意味着这部分空间的缓存命中仍然可以保持。
    • 要增大上述可能性，就要求扩张的倍数能尽量使得前一次释放的内存大小+紧邻的可能空闲的后续内存大小 ≥ 本次要申请的内存大小。而每次 1.5 倍的容量扩张有助于这一点。上述情况是最坏情况。在最好情况下，每次释放的内存恰好是连续分布的，更能轻松满足本次要申请的内存大小，达到复用内存（从而复用已有的可能还未被替换的Cache）的目的。

    什么情况下 vector 迭代器失效？

    • 若超出容量，则全部迭代器失效（因为重新申请内存空间）

    • 末尾插入元素push_back()，未超出容量，则只有finish/vec.end()失效。

      为什么说它会失效？见下面的例子：

      vector<int> vec = {1, 2, 3, 4};
for(auto i = vec.begin(); i != vec.end(); i++){
    vec.push_back(x);
}

      for循环永远不会结束，因为push_back使得vec.end()始终是失效状态，不能正确地指示一个末尾位置。

    • 末尾删除元素，vec.end()失效，原因同上。
    • 中间插入元素insert(pos, n)，未超出容量，则插入点pos及以后的迭代器都失效。

    • 中间删除元素，删除点pos及以后的迭代器都失效，原因同上。

      例如以下代码，执行erase删除第一个 2 后，it自动自增指向第二个2，而后 for 循环执行it++，it指向第三个 2 ，然后删除，并自动自增指向第四个 2， 而后 for 循环执行it++，指向 6

      即循环结束后最终被删除的只有第一和第三个2，仍有 2 未被删除。

      vector<int> nums = {1, 2, 2, 2, 2, 6, 7, 8};
for (vector<int>::iterator it = nums.begin(); it != nums.end(); it++) {
    if (*it == 2) { 
        nums.erase(it); 
    } 
}

    使用迭代器循环，如何安全删除vector中满足给定条件的元素？

    为了让迭代器只通过 for 循环自增，抵消erase的自增操作即可，即：

    vector<int> nums = {1, 2, 2, 2, 2, 6, 7, 8};
for (vector<int>::iterator it = nums.begin(); it != nums.end(); it++) {
    if (*it == 2) { 
        nums.erase(it); 
        it --;
    } 
}
